重置排版 Reboot

重置排版 Reboot

這章節要介紹的是官方文件中 Content 的「Reboot」：
https://getbootstrap.com/docs/4.3/content/reboot/

  • 載入 Bootstrap 就會一起載入 Reboot
  • bs 的 Reboot，就像是 css reset，作用是：將瀏覽器的預設樣式，都設定為一致

:::danger
:heavy_exclamation_mark:
==使用 Bootstrap 時，因為已經有 Reboot 了，就 “不會” 另外載入 CSS Reset==
（Reboot 目的是讓所有瀏覽器均有 “相同的樣式” ）

[補充說明]
CSS reset 有兩種：

  1. 第一種是「完全清除樣式」（例如 meyerweb 的 css reset）
  2. 第二種稱為「normalize」，會加上統一的瀏覽器預設，但並‘不會’完全清除樣式（例如 Bootstrap 的 Reboot）
    :::
  • 「Reboot」章節有兩大重點，分別介紹如下

重點一：box-sizing

Bootstrap 預設載入box-sizing: border-box

先講解 html 在計算「實際呈現的寬度」時的「預設」方法：

  • 有一個寬高都是 300px 的正方形
  • 加入 padding 後，會「向外擴張」
  • 加入 border 後，也會再「向外擴張」
  • 加入 margin 後，又會再「向外擴張」

正方形的「實際呈現的寬度」，會是「width + padding + border」，要加總這麼多東西，非常麻煩！
（margin 不在寬度的計算範圍內）

加入box-sizing: border-box之後

就不用再做複雜的加總了！
:::danger
我設定的寬度 = 最終的寬度（實際呈現的寬度）
:::

不管是加上了 padding 或是 border，都不會影響div「實際呈現的寬度」（300px）

  • 因為 padding、border 會是「向內擴張」
  • margin 依然是「向外擴張」（margin 不在寬度的計算範圍內）

因此，加上 padding、border 後，div的寬度是被壓縮的
（扣掉 padding、border 後，div的寬度 = 180px）

重點二：Native font stack（字體堆疊）

文件網址：
https://getbootstrap.com/docs/4.3/content/reboot/#native-font-stack

內容說明

在 Bootstrap 4 中，已經不再使用「Helvetica Neue, Helvetica, and Arial」做為預設的字體了。而是使用「native font stack」來讓字體在各個裝置和作業系統中，都能有最佳化的呈現。

使用的是$font-family-sans-serif這個變數：

$font-family-sans-serif:
  // Safari for macOS and iOS (San Francisco)
  -apple-system,
  // Chrome < 56 for macOS (San Francisco)
  BlinkMacSystemFont,
  // Windows
  "Segoe UI",
  // Android
  "Roboto",
  // Basic web fallback
  "Helvetica Neue", Arial, sans-serif,
  // Emoji fonts
  "Apple Color Emoji", "Segoe UI Emoji", "Segoe UI Symbol" !default;

:::warning
只要使用 Bootstrap，上面的 font-family 就會被套用在<body>上。

:::

[補充說明] 字體

例如：
Helvetica 是 Mac OS 的預設字體（Windows 並沒有 Helvetica 字體）。

所以 Bootstrap 4 提出了一套新的字體管理方法：
（綠色框框代表：舊版使用的預設字體）

  • 蘋果系統

    • 不管是 pc 還是手機版，使用的字體都是一樣的
    • 預設英文字體：San Francisco
    • 預設中文字體：蘋方（PingFang）

  • Windows 和 Android 都沒有系統預設的字體

Windows 系統的字體問題

:::danger
問題來了：

  • Windows 系統，英文會載入 Segoe UI 和 Roboto 這兩種字體–> 都是「黑體」
  • Windows 系統預設的中文字體是「新細明體」

導致「英文是黑體，中文卻是明體」的問題，看起來很奇怪
:::

建議解決方法

在使用 bs 4 時，可以將「Microsoft JhengHei」加入到 css 的字體裡面，讓字體的協調性更為一致

:::danger
==加到 css 時，不要使用「微軟正黑體」，要用「Microsoft JhengHei」==

這樣才能確保所有瀏覽器都可以正確載入
:::

在 css 載入字體

在載入字體時，分為三個部分

  1. 第一部分：蘋果的預設字體
  • 也就是「系統預設字體」（包含中文、英文字體）
  1. 第二部分：指定已知的系統 UI 字體
  • 只有英文字體，沒有中文（因此要自己指定）
  1. 第三部分：指定「備援的字體」
  • sans-serif 是「無襯線字體」

如何在 css 加入 Microsoft JhengHei 呢？

'Microsoft JhengHei'加入的位置是「第二部分：指定已知的系統 UI 字體」

將font-family寫在 css 中，如下：

:::success

此段加在自訂的 css 即可，也就是另外新增一個 css 檔案加在 bootstrap.min.css 後方即可（就可以覆蓋掉 bs 原先的設定）
:::

body{
    font-family: -apple-system, BlinkMacSystemFont, 'Microsoft JhengHei', 'Segoe UI', Roboto, 'Helvetica Neue', Arial, sans-serif, 'Apple Color Emoji', 'Segoe UI Emoji', 'Segoe UI Symbol';
}

A:
這兩者我無法確認是否相同 (我不是使用 Windows 系統)
而我是依據微軟的設定方法
可以參考相關旗下產品：https://azure.microsoft.com/zh-tw/
因此我還是建議使用 Microsoft JhengHei

文件連結

上面的文件中，「表單」有提到：

:::info

<label> 被設定為display: inline-block 以便讓 margin 應用。
:::

請問這是什麼意思呢？

A:
<label>預設是display: inline
==inline 預設是沒有垂直高，設定margin的話，

需要轉為 inline-block 後，「垂直方向」的margin才有效果。

文字與語意化

Posted on 2019-06-11 | In Bootstrap 4
tags: Bootstrap 4 基礎內容文件要點說明

Section 4-基礎內容文件要點說明

文字與語意化

文件網址：

https://getbootstrap.com/docs/4.3/content/typography/

這章節要講解的是 Content 的「Typography 文字排版」

  • 這章節最大的陷阱在於「標籤的語意」

標籤的語意

標籤的語意，分為兩個部分

  • 第一部分：含語意的標籤
  • 第二部分：不含語意，但具有樣式的 Class

以下分別說明：

第一部分：含語意的標籤

例如：<h1>是有語意的，代表「標題」

html:

<!-- 第一部分：含語意的標籤 -->
    <h1>這是一個標題</h1>

第二部分：不含語意，但具有樣式的 Class

例如：我需要一行字，它要跟 h1 一樣大，但不具有「標題」的語意

  • 跟 h1 一樣大，但它只是一個 Class
<!-- 第二部分：不含語意，但具有樣式的 Class -->
    <p class="h1">這不是一個標題</p>

:::danger
可以看到，<p class="h1"></p>就跟h1一樣大（這個 p 段落，具有跟 h1 一樣的樣式）


:::

Display headings（不含語意，但具有樣式的 Class）

文件網址：
https://getbootstrap.com/docs/4.3/content/typography/#display-headings

如果覺得原本的標題不夠大，可以使用 Display headings 的樣式

範例

我覺得原本的 h1 標題不夠大，所以我在<h1>加上"display-1"這個 class，就會有超級大的樣式了

html:

<!-- 使用 Display headings 的樣式 -->
    <h1 class="display-1">六角學院</h1>

Inline text elements 行內文字元素

文件網址：
https://getbootstrap.com/docs/4.3/content/typography/#inline-text-elements

行內文字元素，是插在整個文字段落中的一個標籤

  • 不會造成任何的「斷行」

範例一：<mark></mark>

<mark></mark>代表「被 highlight 起來」的文字

<p>You can use the mark tag to <mark>highlight</mark> text.</p>

範例二：一些文字樣式

    <p><del>This line of text is meant to be treated as deleted text.(被刪除的內容)</del></p>
    <p><s>This line of text is meant to be treated as no longer accurate.(不準確的內容)</s></p>
    <p><ins>This line of text is meant to be treated as an addition to the document.(補充的內容)</ins></p>
    <p><u>This line of text will render as underlined(下底線的效果)</u></p>
    <p><small>This line of text is meant to be treated as fine print.(縮小的文字)</small></p>

範例三：<strong></strong>與<b></b>的不同

  • ==<strong></strong>是「有強調語意」的標籤==
  • <b></b>跟<strong></strong>長得一樣，但是，==<b></b>‘並沒有’強調「語意的重要性」==
    <p><strong>This line rendered as bold text.(強調的粗體文本)</strong></p>
    <p><b>This line rendered as bold text.(只會突出顯示，但並沒有增加重要性)</b></p>

範例四：<i></i>與<em></em>的不同

  • <i></i>是用在「語音、技術用語」
  • <em></em>跟<i></i>長得一樣，但是，<em></em>只是單純的斜體字
    <p><i>This line rendered as italicized text.(用在語音、技術用語)</i></p>
    <p><em>This line rendered as italicized text.(只是單純的斜體字)</em></p>

範例五：<span></span>

  • <span></span>是行內元素的群組，沒有帶任何語意
<p><span>SPAN 則是行內元素的群組，沒有帶任何語意</span></p>

Abbreviations 縮寫

文件網址：
https://getbootstrap.com/docs/4.3/content/typography/#abbreviations

  • 標籤上會帶一個title（滑鼠移上去時會顯示title）
<p><abbr title="HyperText Markup Language">HTML</abbr></p>

Blockquote 引用的文字

文件網址：
https://getbootstrap.com/docs/4.3/content/typography/#blockquotes

Blockquote 會用在哪裡呢？

  • 在網頁上，引用其他文章、論文的章節內容時

用法：

  • 將「要引用的內容」用<blockquote class="blockquote"></blockquote>包起來

Naming a source 標明出處（是由誰說的）

  • 使用<footer class="blockquote-footer"></footer>來「標明引用內容的出處」
  • 將「出處的來源」用<cite></cite>包起來
    <blockquote class="blockquote">
      <p class="mb-0">霹靂卡霹靂拉拉 波波力那貝貝魯多。</p>
      <footer class="blockquote-footer">小魔女 <cite title="Source Title">Do Re Mi</cite></footer>
    </blockquote>

Lists 清單

文件網址：
https://getbootstrap.com/docs/4.3/content/typography/#lists

  • 使用ul li，預設都會有「圓點」

清除「預設的」標籤樣式

  • 加上class="list-unstyled"，就可以將預設的標籤樣式給清除掉

          <ul class="list-unstyled">
          <li>Lorem ipsum dolor sit amet</li>
          <li>Consectetur adipiscing elit</li>
          <li>Integer molestie lorem at massa</li>
              <ul>
                  <li>Phasellus iaculis neque</li>
                  <li>Purus sodales ultricies</li>
                  <li>Vestibulum laoreet porttitor sem</li>
              </ul>
          </li>
          <li>Faucibus porta lacus fringilla vel</li>
          <li>Aenean sit amet erat nunc</li>
      </ul>

  • 前面沒有「圓點」了

Inline 行內

文件網址：
https://getbootstrap.com/docs/4.3/content/typography/#inline

:::danger
Inline 的作用是：
想要保留ul li（清單）的語意，但是要在「一行」內，把語意表達完
:::

  • 使用class="list-inline"和class="list-inline-item"，就可以把這三個li的內容，用inline的方式呈現（在「一行」內呈現完）
<p>文藝復興三巨星：</p>
    <ul class="list-inline">
      <li class="list-inline-item"><a href="#">達文西</a></li>
      <li class="list-inline-item"><a href="#">米開朗基羅</a></li>
      <li class="list-inline-item"><a href="#">拉斐爾</a></li>
    </ul>
  • 三個li都在同一行

Description list alignment 描述型清單

文件網址：
https://getbootstrap.com/docs/4.3/content/typography/#description-list-alignment

Description list alignment 會用在哪裡呢？

  • 通常用在解釋名詞時
    :::info
  • <dl></dl> 是整個描述型清單的「最外框」
  • <dt></dt> 代表「左側的名詞」（description terms）
  • <dd></dd>代表「右側的描述」（description description）

:heavy_exclamation_mark:
dt跟dd的class="col-sm-*"寬度數值，加起來一定要等於 12，不然會跑版（這個觀念與格線系統是一致的）
:::

將多餘的文字裁切掉 class="text-truncate"

:::warning

其中，有一個很特別的 class 叫做class="text-truncate"，作用是：將多餘的文字裁切掉

  • text-truncate 的使用沒有任何限制：不只能使用在dl dt dd上，也可以用在table上

可以參考以下連結：
https://bootstrap.hexschool.com/docs/4.1/utilities/text/#text-wrapping-and-overflow
:::

        <dl class="row">
            <dt class="col-sm-3">Description lists</dt>
            <dd class="col-sm-9">A description list is perfect for defining terms.</dd>

            <dt class="col-sm-3">Euismod</dt>
            <dd class="col-sm-9">
                <p>Vestibulum id ligula porta felis euismod semper eget lacinia odio sem nec elit.</p>
                <p>Donec id elit non mi porta gravida at eget metus.</p>
            </dd>

            <dt class="col-sm-3">Malesuada porta</dt>
            <dd class="col-sm-9">Etiam porta sem malesuada magna mollis euismod.</dd>

            <dt class="col-sm-3 text-truncate">Truncated term is truncated</dt>
            <dd class="col-sm-9">Fusce dapibus, tellus ac cursus commodo, tortor mauris condimentum nibh, ut fermentum massa
                justo sit amet risus.</dd>

            <dt class="col-sm-3">Nesting</dt>
            <dd class="col-sm-9">
                <dl class="row">
                    <dt class="col-sm-4">Nested definition list</dt>
                    <dd class="col-sm-8">Aenean posuere, tortor sed cursus feugiat, nunc augue blandit nunc.</dd>
                </dl>
            </dd>
        </dl>
  • 下圖中的紅色框框處，就是加上了class="text-truncate"

Responsive font sizes 響應式文字排版

文件網址：
https://getbootstrap.com/docs/4.3/content/typography/#responsive-font-sizes

雖然這段是在 Sass 中運作，
但也可以使用 Media Query 的方式啟用。
範例如下：

@media (max-width: 768px) {
  html {
    font-size: 1.2rem;
  }
}
