重置排版 Reboot

這章節要介紹的是官方文件中 Content 的「Reboot」：
https://getbootstrap.com/docs/4.3/content/reboot/

  • 載入 Bootstrap 就會一起載入 Reboot
  • bs 的 Reboot，就像是 css reset，作用是：將瀏覽器的預設樣式，都設定為一致

:::danger
:heavy_exclamation_mark:
==使用 Bootstrap 時，因為已經有 Reboot 了，就 “不會” 另外載入 CSS Reset==
（Reboot 目的是讓所有瀏覽器均有 “相同的樣式” ）

[補充說明]
CSS reset 有兩種：

  1. 第一種是「完全清除樣式」（例如 meyerweb 的 css reset）
  2. 第二種稱為「normalize」，會加上統一的瀏覽器預設，但並‘不會’完全清除樣式（例如 Bootstrap 的 Reboot）
    :::
  • 「Reboot」章節有兩大重點，分別介紹如下

重點一：box-sizing

Bootstrap 預設載入box-sizing: border-box

先講解 html 在計算「實際呈現的寬度」時的「預設」方法：

  • 有一個寬高都是 300px 的正方形
  • 加入 padding 後，會「向外擴張」
  • 加入 border 後，也會再「向外擴張」
  • 加入 margin 後，又會再「向外擴張」

正方形的「實際呈現的寬度」，會是「width + padding + border」，要加總這麼多東西，非常麻煩！
（margin 不在寬度的計算範圍內）

加入box-sizing: border-box之後

就不用再做複雜的加總了！
:::danger
我設定的寬度 = 最終的寬度（實際呈現的寬度）
:::

不管是加上了 padding 或是 border，都不會影響div「實際呈現的寬度」（300px）

  • 因為 padding、border 會是「向內擴張」
  • margin 依然是「向外擴張」（margin 不在寬度的計算範圍內）

因此，加上 padding、border 後，div的寬度是被壓縮的
（扣掉 padding、border 後，div的寬度 = 180px）

重點二：Native font stack（字體堆疊）

文件網址：
https://getbootstrap.com/docs/4.3/content/reboot/#native-font-stack

內容說明

在 Bootstrap 4 中，已經不再使用「Helvetica Neue, Helvetica, and Arial」做為預設的字體了。而是使用「native font stack」來讓字體在各個裝置和作業系統中，都能有最佳化的呈現。

使用的是$font-family-sans-serif這個變數：

$font-family-sans-serif:
  // Safari for macOS and iOS (San Francisco)
  -apple-system,
  // Chrome < 56 for macOS (San Francisco)
  BlinkMacSystemFont,
  // Windows
  "Segoe UI",
  // Android
  "Roboto",
  // Basic web fallback
  "Helvetica Neue", Arial, sans-serif,
  // Emoji fonts
  "Apple Color Emoji", "Segoe UI Emoji", "Segoe UI Symbol" !default;

:::warning
只要使用 Bootstrap，上面的 font-family 就會被套用在<body>上。

:::

[補充說明] 字體

例如：
Helvetica 是 Mac OS 的預設字體（Windows 並沒有 Helvetica 字體）。

所以 Bootstrap 4 提出了一套新的字體管理方法：
（綠色框框代表：舊版使用的預設字體）

  • 蘋果系統

    • 不管是 pc 還是手機版，使用的字體都是一樣的
    • 預設英文字體：San Francisco
    • 預設中文字體：蘋方（PingFang）

  • Windows 和 Android 都沒有系統預設的字體

Windows 系統的字體問題

:::danger
問題來了：

  • Windows 系統，英文會載入 Segoe UI 和 Roboto 這兩種字體–> 都是「黑體」
  • Windows 系統預設的中文字體是「新細明體」

導致「英文是黑體，中文卻是明體」的問題，看起來很奇怪
:::

建議解決方法

在使用 bs 4 時，可以將「Microsoft JhengHei」加入到 css 的字體裡面，讓字體的協調性更為一致

:::danger
==加到 css 時，不要使用「微軟正黑體」，要用「Microsoft JhengHei」==

這樣才能確保所有瀏覽器都可以正確載入
:::

在 css 載入字體

在載入字體時，分為三個部分

  1. 第一部分：蘋果的預設字體
  • 也就是「系統預設字體」（包含中文、英文字體）
  1. 第二部分：指定已知的系統 UI 字體
  • 只有英文字體，沒有中文（因此要自己指定）
  1. 第三部分：指定「備援的字體」
  • sans-serif 是「無襯線字體」

如何在 css 加入 Microsoft JhengHei 呢？

'Microsoft JhengHei'加入的位置是「第二部分：指定已知的系統 UI 字體」

將font-family寫在 css 中，如下：

:::success

此段加在自訂的 css 即可，也就是另外新增一個 css 檔案加在 bootstrap.min.css 後方即可（就可以覆蓋掉 bs 原先的設定）
:::

body{
    font-family: -apple-system, BlinkMacSystemFont, 'Microsoft JhengHei', 'Segoe UI', Roboto, 'Helvetica Neue', Arial, sans-serif, 'Apple Color Emoji', 'Segoe UI Emoji', 'Segoe UI Symbol';
}

Q&A

Q: 我查了 Windows fonts 資料夾中的字體，發現有 Microsoft JhengHei UI(多了UI兩個字)，但沒有 Microsoft JhengHei，不知 Microsoft JhengHei UI 跟老師提的 Microsoft JhengHei 是同一個字體嗎？

A:
這兩者我無法確認是否相同 (我不是使用 Windows 系統)
而我是依據微軟的設定方法
可以參考相關旗下產品：https://azure.microsoft.com/zh-tw/
因此我還是建議使用 Microsoft JhengHei

Q: Reboot Forms 問題

文件連結

上面的文件中，「表單」有提到：

:::info

<label> 被設定為display: inline-block 以便讓 margin 應用。
:::

請問這是什麼意思呢？

A:
<label>預設是display: inline
==inline 預設是沒有垂直高，設定margin的話，
只有「水平方向」的margin有效果，「垂直方向」的 margin 是沒有效果的。==
需要轉為 inline-block 後，「垂直方向」的margin才有效果。
