為什麼需要陣列？

Posted on 2020-05-03 In JavaScript

什麼情況下會需要使用到「陣列」呢？

在開發時，當我們手邊有一大堆的資料，如果一個一個的宣告變數將會非常費時費力。

例如，我要記錄「農場裡有什麼東西」，就要建立很多個變數：

如果我列了非常多個變數，當我在用 function 去傳遞訊息時，在 function 就必須要帶入「非常多個變數」，這樣很沒有效率。

該如何用「更有效率、更好維護」的方式，去記錄我的資料呢？

🎃 可以透過「陣列」來把資料整理起來

🎃 使用「陣列、物件」，來把「整組的資料」有效率的傳遞到 function 裡面

🎃 「陣列、物件」比「變數」更加的靈活、好用

  • 陣列（array）
  • 物件（object）
